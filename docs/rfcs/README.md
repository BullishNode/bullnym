# RFCs

RFCs are proposals, not documentation of shipped behavior. Each RFC must state
its status, owners, affected repositories, security/recovery invariants,
migration plan, verification plan, and acceptance criteria. Accepted decisions
should be recorded as ADRs; completed implementation detail should be merged
into maintained docs and the RFC archived when it no longer aids maintenance.

- [001: Chain-swap reliability](001-chain-swap-reliability.md)
- [002: Public name reservation](002-public-name-reservation.md)
- [003: Permanent public names](003-permanent-public-names.md)
- [Template](template.md)

RFC-001 and RFC-002 are historical/superseded design records, retained for
rationale rather than active implementation guidance. Current contracts are
linked from their status notices.
